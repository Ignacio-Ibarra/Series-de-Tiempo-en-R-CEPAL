Estimación

En primer lugar seleccionamos la cantidad de rezagos mediante VARselect()[["selection"]]. El parámetro “const” simplemente indica que las ecuaciones del modelo tienen intercepto. Por defecto el máximo orden en 10.

VARselect(cbind(dtasa_polmon,dprestamos_personales), type="const")[["selection"]]
## AIC(n)  HQ(n)  SC(n) FPE(n) 
##     10      6      1      9

Vemos que el criterio AIC sugiere 10 rezagos, mientras que el BIC 1. Realizamos la estimación con 1, por lo que el modelo a estimar, llamando \(TPM\) a la tasa de política monetaria y \(PP\) a la de préstamos personales, es:

\[\begin{aligned} \Delta TPM_t = \beta_{10}&+ \beta_{11} \ \Delta TPM_{t-1}+ \beta_{12} \ \Delta PP_{t-1} + \varepsilon_{1t} \\ \Delta PP_t = \beta_{20}&+ \beta_{21} \ \Delta PP_{t-1} + \beta_{22} \ \Delta TPM_{t-1} + \varepsilon_{2t} \end{aligned}\]

Para estimar el modelo usamos la función VAR(), agregando una constante y con 1 rezago (p=1). Mediante summary() podemos evaluar la significatividad estadística de los coeficientes, pero no pueden interpretarse de la misma manera que las regresiones que vimos hasta ahora porque no asumimos exogeneidad, sino que todas las variables son endógenas y simétricas. Por el momento, lo que interesa es el resultado de la proyección.

var <- VAR(cbind(dtasa_polmon,dprestamos_personales), p=1, type="const")

Pronóstico

Para hacer el pronóstico manualmente necesitaríamos acceder a los coeficientes, por ejemplo mediante coef(). Como en un VAR hay más de una ecuación, los coeficientes se guardan en una lista, cada uno de cuyos elementos corresponde a una ecuación. Podemos acceder a cada elemento de una lista con corchetes dobles.

class(coef(var))
## [1] "list"
coef_polmon <- coef(var)[[1]]
coef_prestamos <- coef(var)[[2]]

La función forecast() permite generar un pronóstico automáticamente, lo cual es posible porque las variables en \(t+1\) dependen exclusivamente de su valor en \(t\), que es conocido. En particular, no hay variables exógenas cuyos valores futuros haya que pre-determinar para hacer el pronóstico (como hicimos, por ejemplo, con el MCE). El parámetro h fija el horizonte de la proyección.

autoplot(forecast(var, h=6)) + xlab("Year")

Para acceder a los valores pronosticados entramos al objeto forecast dentro del pronóstico. Es una lista cuyo primer elemento es el pronóstico para dtasa_polmon, y el segundo para dprestamos_personales. Las estimaciones puntuales se guardan bajo el nombre de mean.

Podríamos replicar el pronóstico para la variable en niveles sumándole al valor conocido en \(t\) la diferencia pronosticada para \(t+1\), e iterando. Por ejemplo, para la tasa de política monetaria calculamos:

\[ \begin{aligned} TPM_{t+1}^p&=TPM_{t}+\Delta TPM_{t+1}^p \end{aligned}\]

donde el supraíndice p indica los valores pronosticados. Para \(t=48\) tasa_polmon es conocida, mientras que para los períodos posteriores es el valor pronosticado (por ser una proyección dinámica).

  • Guardamos las diferencias pronosticadas en los períodos sucesivos a la serie original.
  • Creamos las variables donde vamos a cargar el pronóstico.
dtasa_polmon_p <- c(dtasa_polmon, dtasa_polmon_p)
dprestamos_personales_p <- c(dprestamos_personales, dprestamos_personales_p)

tasa_polmon_p <- tasa_polmon
prestamos_personales_p <- prestamos_personales

Ahora, usando un loop, construimos el pronóstico sumando a cada variable en \(t-1\) la diferencia pronosticada para el período \(t\), lo cual nos devuelve el nivel pronosticado en \(t\).

for (t in 49:54) {
tasa_polmon_p <- c(tasa_polmon_p,
                     tasa_polmon_p[t-1]+dtasa_polmon_p[t])
prestamos_personales_p <- c(prestamos_personales_p,
                            prestamos_personales_p[t-1]+dprestamos_personales_p[t])
}

Este modelo ajusta peor a lo que hoy sabemos que sucedió que el MCE porque en este último incorporamos conocimiento sobre la variable de política. Esta metodología VAR es “agnóstica” respecto a qué variables son endógenas y cuáles no, por lo que aplica mejor cuando esperamos que estas mantengan su trayectoria (conjunta) histórica. También podrían incorporarse shocks exógenos sobre algunas de estas variables endógenas. Si bien no podemos verlo en detalle, estos shocks pueden visualizarse a través de lo que se conoce como función impulso-respuesta, de la siguiente manera:

#Forecast error impulse response

feir <- irf(var, impulse = "dtasa_polmon", response = "dprestamos_personales",
             n.ahead = 8, ortho = FALSE, runs = 1000)

plot(feir)

Notar que el primer valor es cero porque no hay efectos contemporáneos en ninguna de las ecuaciones. Para incorporar efectos contemporáneos entre las variables habría que emplear estrategias como la descomposición de Cholesky, donde se rompe la relación de simetría haciendo supuestos de identificación sobre el orden de exogeneidad a las variables. Para más detalles se sugiere ver el libro de Enders.

Selección de modelos de pronóstico alternativos

Supongamos nuevamente que no contamos con información para períodos posteriores a diciembre de 2019, y que queremos evaluar dos modelos alternativos para pronosticar la tasa de préstamos personales en un contexto donde todas las variables son endógenas. Para estimar un VAR, como vimos, necesitamos que las series sean estacionarias. Para eso, tomamos primeras diferencias.

\[\begin{aligned} \Delta PP_t &= c_1 +\beta_1 \ \Delta PP_t +\epsilon_t \\ PP_t &= c_1 +\beta_1 \ \Delta PP_t +\beta_2 \ \Delta \text{Liquidez}_t+\varepsilon_t \end{aligned}\]

donde \(\text{Liquidez}_t\) es la liquidez bancaria, calculada como la suma de la cuenta corriente en el BC, las LELIQs y los pases pasivos sobre los depósitos (también descargados de seriese.xlsx del BCRA). La variable se encuentra disponible, ya mensualizada, en tasas.

autoplot(liquidez)

Notemos que la serie es integrada de orden 1.

Siguiendo con la evaluación, dividimos la muestra en la de entrenamiento (una para cada modelo) y de prueba (solo la variable que nos interesa pronosticar). window() permite recortar las series según los parámetros especificados, a diferencia de ts(), que solo declara el período de la serie.

entrenamiento1 <- window(cbind(dprestamos_personales,dtasa_polmon), 
                         start = c(2016,2), end = c(2019,4))
entrenamiento2 <- window(cbind(dprestamos_personales,dtasa_polmon,dliquidez), 
                         start = c(2016,2), end = c(2019,4))

prueba <- window(dprestamos_personales, start = c(2019,5), end = c(2019,12), frequency = 12)

Ahora estimamos cada uno de los modelos y guardamos el pronóstico. Para la selección se rezagos máximos, tener en cuenta que se pierden muchos grados de libertad y la muestra no es tan grande: con 3 rezagos en el VAR de 2 variables perdemos 8, con 2 rezagos en el VAR de dimension 3 perdemos 9 (aprox. 20% de la muestra). Fijamos un horizonte de pronóstico de 8 períodos, que es el tamaño de la muestra de prueba.

VARselect(entrenamiento1, lag.max = 3, type = "const")[["selection"]] 
## AIC(n)  HQ(n)  SC(n) FPE(n) 
##      1      1      1      1
VARselect(entrenamiento2, lag.max = 2, type = "const")[["selection"]] 
## AIC(n)  HQ(n)  SC(n) FPE(n) 
##      1      1      1      1
var1 <- forecast(VAR(entrenamiento1, p=1,type="const"),h=8)
var2 <- forecast(VAR(entrenamiento2, p=1,type="const"),h=8)

Recuperamos los pronósticos para la (diferencia de la) tasa de préstamos personales.

dprestamos_personales_adj1 <- var1$forecast$dprestamos_personales$mean
dprestamos_personales_adj2 <- var2$forecast$dprestamos_personales$mean

prestamos_personales_adj2 <- prestamos_personales
